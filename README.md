## 项目简介

本项目是一个**智能化的基金经理风格分析系统**，专门针对**二级债基**和**可转债基金**进行深度分析。系统利用大语言模型（DeepSeek-V3.2）对基金过去三年的定期报告进行文本挖掘，构建了一套涵盖**收益风险、操作风格、个性特征**三大维度的客观、可量化标签体系与多维画像。

### 应用场景

- 🏦 **机构投资者**：快速筛选符合投资目标的基金经理
- 📊 **基金研究**：深入分析基金经理投资风格与能力圈
- 🔍 **尽职调查**：辅助投资决策与风险评估
- 📈 **组合构建**：识别风格互补的基金经理进行组合配置

## 核心特性

| 特性                       | 说明                                                       |
| -------------------------- | ---------------------------------------------------------- |
| 🤖**LLM驱动分析**    | 使用DeepSeek大语言模型进行深度语义理解，而非简单关键词匹配 |
| 🎯**基金类型区分**   | 自动识别并区分二级债基、可转债基金，采用差异化评价标准     |
| 📊**双轨制评价**     | 量化指标计算 + 定性标签转换，兼顾客观性与可读性            |
| ⏱️**多期时序分析** | 分析多个报告期，通过多数投票提取穿越周期的长效风格         |
| 🔄**智能降级机制**   | LLM不可用时自动降级为关键词匹配，保证系统稳定性            |
| 💰**成本可控**       | 单个基金分析成本约0.055-0.11元，适合大规模应用             |

## 快速开始

### 环境要求

- Python 3.8+
- DeepSeek API Key

### 安装步骤

```bash
# 1. 克隆项目
git clone https://github.com/your-repo/LLM-Driven-Behavioral-and-Risk-Tagging-System-for-FIP-Funds.git
cd LLM-Driven-Behavioral-and-Risk-Tagging-System-for-FIP-Funds

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置环境变量
# 创建 .env 文件并填入以下内容：
```

```env
DEEPSEEK_API_KEY=your_api_key_here
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
DEEPSEEK_MODEL=deepseek-chat
DEEPSEEK_TEMPERATURE=0.3
DEEPSEEK_MAX_TOKENS=1000
```

### 运行分析

```bash
# 执行主程序
python project/main.py
```

### 数据准备

1. **基金报告PDF**：放置于 `data/fund_reports/` 目录
2. **基金净值数据**：放置于 `data/fund_nav/` 目录
3. **基金列表**：编辑 `input/筛选后的基金列表.xlsx`

## 核心模块

### 1. 数据提取模块 (`data_extraction`)

负责从PDF定期报告中提取结构化数据：

| 组件                          | 功能                   | 输出                    |
| ----------------------------- | ---------------------- | ----------------------- |
| `base_extractor.py`         | PDF文本解析、章节识别  | 原始文本                |
| `macro_view_extractor.py`   | 投资策略、宏观展望提取 | 文本段落                |
| `performance_extractor.py`  | 业绩指标提取           | 收益率、回撤等          |
| `holding_extractor.py`      | 持仓结构提取           | 资产配置、股票/债券持仓 |
| `manager_info_extractor.py` | 基金经理信息提取       | 履历、管理经验          |

### 2. 标签生成模块 (`tag_generation`)

基于提取的数据生成多维度标签：

| 组件                               | 功能         | 数据来源            |
| ---------------------------------- | ------------ | ------------------- |
| `risk_return_tagger.py`          | 收益风险标签 | 净值数据            |
| `operation_style_tagger.py`      | 操作风格标签 | 持仓数据 + 策略文本 |
| `personality_tagger.py`          | 个性特征标签 | 持仓数据 + 策略文本 |
| `multi_period_style_analyzer.py` | 多期风格聚合 | 短期标签序列        |

### 3. 配置模块 (`config`)

定义不同基金类型的特性与评价标准：

```python
# 基金类型配置示例
FUND_TYPE_CONFIG = {

            "债券": (0.70, 0.90)

        "risk_return_characteristics": "中低风险中收益",
        ...
    },
    "CONVERTIBLE_BOND": {    # 可转债基金
        "asset_allocation": {
```

## 标签体系

### 📊 收益风险标签

基于净值数据计算的量化指标，转换为定性标签：

| 标签类型       | 指标               | 标签值示例                             |
| -------------- | ------------------ | -------------------------------------- |
| 长期收益能力   | 近一/二/三年收益率 | 高收益特征 / 中等收益特征 / 低收益特征 |
| 回撤控制水平   | 最大回撤           | 回撤控制极佳 / 较强 / 中等 / 较弱      |
| 业绩波动特征   | 年化标准差         | 低波稳健 / 弹性中波 / 高波弹性         |
| 风险调整后收益 | 夏普比率           | 性价比优秀 / 良好 / 一般 / 较差        |
| 超额获取能力   | 相对基准超额收益   | 长期跑赢基准 / 持平 / 跑输基准         |
| 胜率特征       | 日/周/月胜率       | 高胜率 / 一般胜率 / 低胜率             |

### 🎯 操作风格标签

基于持仓变化与市场走势的关系判断：

| 标签值                 | 含义     | 特征描述                   |
| ---------------------- | -------- | -------------------------- |
| **强左侧特征**   | 逆向投资 | 市场下跌时加仓，上涨时减仓 |
| **中等左侧特征** | 偏逆向   | 倾向于左侧布局，但不过度   |
| **均衡特征**     | 灵活配置 | 左右侧结合，根据市场调整   |
| **中等右侧特征** | 偏趋势   | 倾向于右侧跟随，适度左侧   |
| **强右侧特征**   | 趋势跟随 | 市场上涨时加仓，下跌时减仓 |

### 🏷️ 个性特征标签

基于持仓结构与投资策略文本分析：

| 维度               | 标签值                                      | 说明               |
| ------------------ | ------------------------------------------- | ------------------ |
| **信用下沉** | 积极信用下沉 / 适度信用下沉 / 高等级为主    | 信用债配置偏好     |
| **权益风格** | 价值风格 / 成长风格 / 平衡风格 / 无权益持仓 | 股票/转债风格偏好  |
| **回撤控制** | 极强 / 较强 / 中等 / 较弱                   | 回撤控制能力与意愿 |
| **利率波段** | 积极波段操作 / 适度波段操作 / 持有到期为主  | 利率债交易频率     |

## 使用指南

### 完整分析流程

```python
# 1. 数据提取阶段
# 扫描 data/fund_reports/ 目录下的所有PDF文件
# 提取宏观观点、业绩数据、持仓数据
# 计算数据完整度得分
# 导出MD格式的提取结果

# 2. 标签生成阶段
# - 收益风险标签：基于净值数据计算
# - 操作风格标签：LLM分析 + 持仓验证
# - 个性特征标签：LLM分析 + 持仓验证
# - 多期聚合：短期标签 → 多数投票 → 长期标签

# 3. 画像生成阶段
# 整合所有标签，生成基金经理画像描述
```

### 单独使用各模块

```python
# 数据提取示例
from project.data_extraction import MacroViewExtractor, HoldingExtractor

macro_extractor = MacroViewExtractor()
macro_data = macro_extractor.extract(pdf_text, report_type)

holding_extractor = HoldingExtractor()
holding_data = holding_extractor.extract(pdf_text, report_type)

# 标签生成示例
from project.tag_generation import RiskReturnTagger, OperationStyleTagger

risk_tagger = RiskReturnTagger(fund_type="SECONDARY_BOND")
risk_tags = risk_tagger.generate(nav_data)

operation_tagger = OperationStyleTagger()
operation_tags = operation_tagger.generate(holding_data, macro_data, index_data)
```

### 环境变量配置

| ------------------------ | ---------------- | ------------------------------- |
| `DEEPSEEK_API_KEY`     | DeepSeek API密钥 | 必填                            |
| `DEEPSEEK_BASE_URL`    | API基础URL       | `https://api.deepseek.com/v1` |

| `DEEPSEEK_TEMPERATURE` | 生成温度         | `0.3`                         |
| `DEEPSEEK_MAX_TOKENS`  | 最大生成长度     | `1000`                        |

### 基金类型配置

系统支持三种基金类型的差异化分析：

| 基金类型   | 代码                 | 资产配置特征           | 风险收益特征     |
| ---------- | -------------------- | ---------------------- | ---------------- |
| 二级债基   | `SECONDARY_BOND`   | 股票≤20%，债券≥70%   | 中低风险中收益   |
| 可转债基金 | `CONVERTIBLE_BOND` | 可转债≥60%            | 中高风险中高收益 |
| 纯债型基金 | `PRIMARY_BOND`     | 债券≥80%，无股票/转债 | 低风险低收益     |

## 常见问题

### Q1: 支持哪些基金类型？

目前支持**二级债基**和**可转债基金**两种类型。系统会自动识别基金类型并应用相应的评价标准。

### Q2: 需要多少历史报告？

建议至少**6个季度报告**（约1.5年）才能获得较稳定的风格判断。系统支持分析任意数量的报告。

### Q3: 如何处理LLM调用失败？

系统内置**智能降级机制**：当LLM不可用时，自动降级为关键词匹配方法，保证系统稳定性。

### Q4: 输出结果如何使用？

- **JSON文件**：可用于后续量化分析、数据库存储
- **画像描述**：可直接用于报告撰写、投资决策参考
- **汇总报告**：快速了解所有基金的分析结果

### Q5: 如何扩展新的基金类型？

在 `project/config/fund_type_characteristics.py` 中添加新的基金类型配置，包括资产配置限制、收益风险特征、Prompt上下文等。
